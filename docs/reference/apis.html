APIs

Overview​

Nakji offers two APIs, a REST (historical) API as well as a Realtime API.

This is a work in progress. If you have any questions and concerns, please feel free to reach out to us via TG, Discord, or email.

REST API​

The REST API is for getting metadata and historical data. All REST API endpoints require authentication during the private beta.

The endpoints starting with /tv are designed to be compatible with the Tradingview charting library. To get access to the Tradingview Charting Library, please refer to the tradingview.com website.

Base URL: https://api.nakji.network/v1

Endpoints​

/tv/reference/symbols​

List all available streams. This endpoint corresponds with searchsymbol() in the Tradingview Charting Library.

note

This response may take some time to finish loading due to the large amount of streams.

Request vars​

  • search: search for contract address, token name, etc.
  • namespace: retrieve the most recently active pairs in a protocol

Response vars​

  • ticker: unique identifier for the stream. This is the only variable you need if not using the Tradingview charting library. We refer to this as streamName.
  • symbol: Human readable name of this data stream
  • full_name: Human readable name of this data stream, including its namespace. Format: Dapp:symbol
  • description: Description for Tradingview API. If applicable, also includes the contract address to disambiguate overlapping symbol names
  • exchange: the name of the DApp or exchange the stream is coming from
  • type: type of the stream. This could be either indicator or aggregate (Nakji aggregates individual trades into candlesticks)

    curl -i -X GET \
      -H 'Origin: https://yourdomain' \
      -H 'Authorization:Bearer <apikey>' \
      "https://api.nakji.network/v1/tv/reference/symbols"

/tv/reference/symbols/<streamName>​

Get full details about a stream. This endpoint corresponds with resolveSymbol() in the Tradingview Charting Library and is usually not necessary for other applications.

Request vars​

0 GET variables. streamName is the name of the stream, and is returned as ticker in the /tv/reference/symbols endpoint (eg. pancakeswapv2.0x1016CFdA658D6Df132e15D2DFdd90F195c22886f.trade-1m0s).

Response vars​

All response vars correspond with Tradingview charting library's symbology and SymbolInfo struct.

/data​

Get historical data. The data format this endpoint returns is the same as it is stored.

Request vars​

  • symbol: search filter (e.g. uniswapv2.0xBB8a43724dE8BC071d7278bAcCB19363545fE758.liquiditypoolevent)
  • from (optional): time window lower bound. In unix time ms.
  • to (optional): time window upper bound, defaults to now. In unix time ms.
  • limit (optional): database query limit (max. 100). In int.
  • resolution (optional): time box, defaults to 1 minute. In enum.
//available resolution enums
{   
    "1S":  "1 second",
    "5S":  "5 seconds",
    "10S": "10 seconds",
    "15S": "15 seconds",
    "30S": "30 seconds",
    "1":   "1 minute",
    "3":   "3 minute",
    "5":   "5 minutes",
    "15":  "15 minutes",
    "30":  "30 minutes",
    "60":  "1 hour",
    "1H":  "1 hour",
    "2H":  "2 hour",
    "4H":  "4 hour",
    "D":   "1 day",
    "W":   "1 week",
}

Response​

The response depends on the stream you are querying, and is determined by the connector from in this data stream was ingested.

/pair​

Retrieves metadata of a pair.

Request vars​

  • network: network name (e.g. ethereum)
  • address: hex address within the network (e.g. 0xBB8a43724dE8BC071d7278bAcCB19363545fE758)
Example​
curl -i -X GET \
    -H 'Origin: https://yourdomain' \
    -H 'Authorization:Bearer <apikey>' \
    "https://api.nakji.network/v1/pair?network=ethereum&address=0x67483a1229b1304d9df461fb8e7671b1bd66e3fa"

Response vars​

  • network: network name (e.g. ethereum)
  • address: hex address within the network (e.g. 0xBB8a43724dE8BC071d7278bAcCB19363545fE758)
  • app: dApp name of where the data is from
  • token0: information on the first token of the pair. Includes address, decimals, name, and symbol
  • token1: information on the second token of the pair. Includes address, decimals, name, and symbol
Example​
{
    "network": "ethereum",
    "address": "0x67483a1229b1304d9df461fb8e7671b1bd66e3fa",
    "app": "uniswapv2",

        "address": "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48",
        "decimals": 6,
        "name": "USD Coin",
        "symbol": "USDC"
    },
    "token1": {
        "address": "0xD2dDa223b2617cB616c1580db421e4cFAe6a8a85",
        "decimals": 18,
        "name": "Bondly Token",
        "symbol": "BONDLY"
    }
}

/lpe/ohlcvvvv​

(/ Liquidity Pool Event / Open High Low Close Volume Volume Volume Volume)

Get candlestick data for any AMM dex that is supported. Only streams that end in .liquiditypoolevent-1m0s or .trade-1m0s can use this endpoint.

Timeframes(resolutions) larger than 1m are aggregated using the *-1m0s data.

Request vars​

  • symbol: hex address within the network (e.g. 0xBB8a43724dE8BC071d7278bAcCB19363545fE758)
  • from: hex address within the network (e.g. 0xBB8a43724dE8BC071d7278bAcCB19363545fE758)
  • to: hex address within the network (e.g. 0xBB8a43724dE8BC071d7278bAcCB19363545fE758)
  • resolution: hex address within the network (e.g. 0xBB8a43724dE8BC071d7278bAcCB19363545fE758)
Example​
curl -i -X GET \
  -H 'Origin: https://yourdomain' \
  -H 'Authorization:Bearer <apikey>' \
"https://api.nakji.network/v1/lpe/ohlcvvvv?symbol=uniswapv2.0xCc3d1EceF1F9fD25599dbeA2755019DC09db3c54.liquiditypoolevent-1m0s&from=1615885305000&to=1616091703000&resolution=1"

Response​

Newlined delimited json, where each line has the following struct/example:

{
  // Timestamp
    "T":"2021-03-16T10:35:00Z",
  
  // OHLC (candlestick)
    "Open":0.0030447661831800705,
    "Close":0.0030447661831800705,
    "High":0.0030447661831800705,
    "Low":0.0030447661831800705,

  // Volumes, where Amount0/Amount1 corresponds to Token0/Token1 in a trading pair.
  // for SRM/ETH, SRM is 0, ETH is 1
  // This gives you the flexibility to choose units as well as show how much volume was bought and sold on each side.
    "Amount0in":0, // SRM sold (in SRM units)
    "Amount0out":133.799515, // SRM bought (in SRM units)
    "Amount1in":0.4073882385978945, // ETH sold (in ETH units)
    "Amount1out":0 // ETH bought (in ETH units)
}

Realtime API​

Base URL: https://stream.nakji.network/

  • /ws: Websockets endpoint.
  • /sse: Server sent events endpoint.
note

Both endpoints only support unidirectional data transmission at this time.

Response data structures depend on the data that is indexed for that specific stream.

Examples​

You can use wscat to test the websockets endpoint from your terminal. You can use curl to query the SSE endpoint.

single stream​

wscat -c "wss://stream.nakji.network/ws?streams=sushiswap.<pairid>.trade-1m0s"
curl https://stream.nakji.network/sse?streams=sushiswap.<pairid>.trade-1m0s

wildcard streams​

# This will subscribe to real time LP updates.
# For example, for all uniswap pairs you can subscribe to `uniswapv2.*.liquiditypool`
wscat -c "wss://stream.nakji.network/ws?streams=sushiswap.*.trade-1m0s"
curl https://stream.nakji.network/sse?streams=sushiswap.*.trade-1m0s

multiple streams​

wscat -c "wss://stream.nakji.network/ws?streams=x&streams=y&streams=z"
curl https://stream.nakji.network/sse?streams=x&streams=y&streams=z

Notes​

Stream names are case sensitive. Pair IDs are case sensitive to their checksummed values.
