APIs

Overviewâ€‹

Nakji offers two APIs, a REST (historical) API as well as a Realtime API.

This is a work in progress. If you have any questions and concerns, please feel free to reach out to us via TG, Discord, or email.

REST APIâ€‹

The REST API is for getting metadata and historical data. All REST API endpoints require authentication during the private beta.

The endpoints starting with /tv are designed to be compatible with the Tradingview charting library.

Base URL: https://api.nakji.network/v1

Endpointsâ€‹

/tv/reference/symbolsâ€‹

List all available streams. Note: this response may take some time to finish loading

Request vars:

  • search: search for contract address, token name, etc.
  • namespace: retrieve the most recently active pairs in a protocol

Example:â€‹

curl -i -X GET \
  -H 'Origin: https://yourdomain' \
  -H 'Authorization:Bearer <apikey>' \
  "https://api.nakji.network/v1/tv/reference/symbols"

/dataâ€‹

Get historical data. The data format this endpoint returns is the same as it is stored.

Request vars:

  • param symbol seearch filter (e.g. uniswapv2.0xBB8a43724dE8BC071d7278bAcCB19363545fE758.liquiditypoolevent)
  • param unix time ms from (optional) time window lower bound
  • param unix time ms to (optional) time window upper bound, defaults to now.
  • param int limit (optional) database query limit (max. 100)
  • param enum resolution (optional) time box, defaults to 1 minute {
    "1S": "1 second", "5S": "5 seconds", "10S": "10 seconds", "15S": "15 seconds", "30S": "30 seconds", "1": "1 minute", "3": "3 minute", "5": "5 minutes", "15": "15 minutes", "30": "30 minutes", "60": "1 hour", "1H": "1 hour", "2H": "2 hour", "4H": "4 hour", "D": "1 day", "W": "1 week", }

Example:â€‹

curl -i -X GET \
  -H 'Origin: https://youtdomain' \
  -H 'Authorization:Bearer <apikey>' \
  "https://api.nakji.network/v1/data?symbol=daofi.0x2C98813dea4AA80F0F160748450f0ABFd51FB558.liquiditypoolevent&from=1616805306000&to=1616915273000&resolution=1"

Request vars:

  • param network network name (e.g. ethereum)
  • param address hex address within the network (e.g. 0xBB8a43724dE8BC071d7278bAcCB19363545fE758)

/lpe/ohlcvvvvâ€‹

Get candlestick data for any AMM dex that is supported. Only streams that end in .liquiditypoolevent-1m0s or .trade-1m0s can use this endpoint. Timeframes(resolutions) larger than 1m are aggregated using the *-1m0s data.

curl -i -X GET \
  -H 'Origin: https://youtdomain' \
  -H 'Authorization:Bearer <apikey>' \
"https://api.nakji.network/v1/lpe/ohlcvvvv?apikey=&symbol=uniswapv2.0xCc3d1EceF1F9fD25599dbeA2755019DC09db3c54.liquiditypoolevent-1m0s&from=1615885305000&to=1616091703000&resolution=1"

Realtime APIâ€‹

Base URL: https://stream.nakji.network/

  • /ws: Websockets endpoint.
  • /sse: Server sent events endpoint.

Note: Both endpoints only support unidirectional data transmission at this time.

Examplesâ€‹

You can use wscat to test the websockets endpoint from your terminal. You can use curl to query the SSE endpoint.

single streamâ€‹

wscat -c "wss://stream.nakji.network/ws?streams=sushiswap.<pairid>.trade-1m0s"
curl https://stream.nakji.network/sse?streams=sushiswap.<pairid>.trade-1m0s

wildcard streamsâ€‹

# This will subscribe to real time LP updates.
# For example, for all uniswap pairs you can subscribe to `uniswapv2.*.liquiditypool`
wscat -c "wss://stream.nakji.network/ws?streams=sushiswap.*.trade-1m0s"
curl https://stream.nakji.network/sse?streams=sushiswap.*.trade-1m0s

multiple streamsâ€‹

wscat -c "wss://stream.nakji.network/ws?streams=x&streams=y&streams=z"
curl https://stream.nakji.network/sse?streams=x&streams=y&streams=z

Notesâ€‹

Stream names are case sensitive. Pair IDs are case sensitive to their checksummed values.
