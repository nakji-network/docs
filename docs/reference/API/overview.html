Overview

For data, Nakji offers two APIs, a REST (historical) API as well as a Realtime API. You can also learn more about what data streams are available on Nakji and details of each data stream through our System Metadata APIs.

This document is a work in progress. If you have any questions, comments, or concerns, please feel free to reach out to us via Telegram, Discord, or email.

Data encoding​

Some data is stored as bytes, and are delivered via the historical and real-time api encoded as base64. API clients should decode from base64 to get their desired format.

For example, to decode from base64 into int in javascript:

parseInt(Buffer.from(rawData, "base64").toString('hex'), 16)

For fields that are typically displayed in hex (eg. Addresses and Transaction hashes), you can convert from base64 encoding to hex. Example in javascript below:

const buffer = Buffer.from(rawData, 'base64');
buffer.toString('hex');

Full example​

The following is a single data point from https://api.nakji.network/v1/data/nakji.common.0_0_0.liquiditypool_change:

const data = {
  "ts": {
    "seconds": 1652507010
  },
  "block": 14771872,
  "idx": 967,
  "tx": "tnS9qc9guCFqLLB/QO0Aoo7BaVeuVN4W01EJpMSevjM=",
  "sender": "3vHA3tm+x/GhZwgZgzJA8CeyXv8=",
  "amount0in": "5OfhaK80AA==",
  "amount1out": "AShqmM6480Tr",
  "to": "dN5dT8v2PgApb9ldMyNrl5QBZjE="
}

You'll notice that tx, sender, to, amount0in, and amount0out are all encoded in base64.

To decode tx, which is expected as a hex value:

const buffer = Buffer.from(data.tx, 'base64');
console.log(buffer.toString('hex'));
// tx: 0xb674bda9cf60b8216a2cb07f40ed00a28ec16957ae54de16d35109a4c49ebe33
// https://etherscan.io/tx/0xb674bda9cf60b8216a2cb07f40ed00a28ec16957ae54de16d35109a4c49ebe33

To decode sender/to, which are expected as a hex values:

console.log(Buffer.from(data.sender, 'base64').toString('hex'));
// sender: 0xdef1c0ded9bec7f1a1670819833240f027b25eff
console.log(Buffer.from(data.to, 'base64').toString('hex'));
// to: 0x74de5d4fcbf63e00296fd95d33236b9794016631

To decode amount0in and amount1out, which are expected to be number values:

parseInt(Buffer.from(data.amount0in, "base64").toString('hex'), 16)
// amount0in: 64431250000000000
parseInt(Buffer.from(data.amount1out, "base64").toString('hex'), 16)
// amount1out: 21359052196391634155
