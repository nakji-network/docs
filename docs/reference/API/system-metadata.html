System Metadata

System Metadata APIs are designed to give you more information about the Nakji Ecosystem. Through the System Metadata APIs, you can get the full list of data streams that are available on Nakji Network and details of specific data streams.

All System Metadata endpoints require authentication during the private beta.

Base URL: https://api.nakji.network/v1

Endpoints​

/metadata/connectors​

Returns a list of all available connectors that are available on Nakji network.

Request vars​

0 GET variables.

Response vars​

An array of strings of connector names.

Example​

  curl -i -X GET \
      -H 'Origin: https://yourdomain' \
      -H 'Authorization:Bearer <apikey>' \
      "https://api.nakji.network/v1/metadata/connectors"

[
  "nakji.aave.0_0_0",
  "nakji.chi.0_0_0",
  "nakji.eth2.0_0_0",
  "nakji.solana.0_0_0",
  "nakji.traderjoe.0_0_0",
  ...
]

/metadata/streams​

Returns a list of all available data streams available on Nakji.

Request vars​

0 GET variables.

Response vars​

An array of JSON objects with the following properties:

  • name: name of the data stream
  • persisted: boolean value that tells you if the historical data for the data stream is available. If false, /data API is not available for the given data stream

Example​

    # list all
    curl -i -X GET \
      -H 'Origin: https://yourdomain' \
      -H 'Authorization:Bearer <apikey>' \
      "https://api.nakji.network/v1/metadata/streams"

    # Response format is `json`
    [
      {
        "name": "nakji.solana.0_0_0.solana_reward",
        "persisted": true
      },
      {
        "name": "nakji.perpetual.0_0_0.stakedperptoken_staked",
        "persisted": false
      },
      {...}
    ]

/metadata/stream/:name/keys​

Given a stream name, return an array of namespace-subject pairs that exists for the given data stream. These can be used as values for the optional request vars for the historical or real-time data endpoints.

note

Not all streams have keys. In those cases, the API will return an empty list. This endpoint may take a while to load when the stream has a lot of historical data (eg nakji.common.0_0_0.*), and is a known issue that the Nakji team is working on. In the meantime, you may subscribe to the stream using websocket api to stream the latest data in real-time, which will include namespace and subject fields in every message.

Request vars​

0 GET variables. :name is the name of the stream. (Eg. nakji.solana.0_0_0.solana_reward).

Response vars​

An array of JSON objects with the following properties:

  • namespace: namespace of the key
  • subject: subject of the key

Example​

    curl -i -X GET \
      -H 'Origin: https://yourdomain' \
      -H 'Authorization:Bearer <apikey>' 
      https://api.nakji.network/v1/metadata/stream/nakji.common.0_0_0.liquiditypool_change/keys


    {
      "namespace": "sushiswap",
      "subject": "0x58Dc5a51fE44589BEb22E8CE67720B5BC5378009"
    }
    {
      "namespace": "pancakeswap",
      "subject": "0x116074C3548C9C761F68850fb8609e90F7BCe791"
    }
    {
      "namespace": "uniswapv2",
      "subject": "0x3d2816aF5A6a0Ac9c62fb9bb51a03723d6EC4c36"
    }
