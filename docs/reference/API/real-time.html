Realtime Data

Realtime Data Websockets are designed to provide realtime data through our data streams. All data streams are available via real-time API.

Base URL: https://stream.nakji.network/

  • /ws: Websockets endpoint.
  • /sse: Server sent events endpoint.

Request vars​

  • streams: stream names to subscribe to (e.g. nakji.common.0_0_0.liquiditypool_change-1m0s)

Additionally, for nakji.common streams:

  • namespace (optional): dApp name (e.g. uniswapv2)
  • subjects (optional): pair addresses within the namespace (e.g. 0x123)
note

Both endpoints only support unidirectional data transmission at this time.

Response data structures depend on the data that is indexed for that specific stream.

Examples​

You can use wscat to test the websockets endpoint from your terminal. You can use curl to query the SSE endpoint.

single stream​

wscat -c "wss://stream.nakji.network/ws?streams=nakji.common.0_0_0.market_trade-1m0s&namespace=sushiswap&subjects=<pairid>"
curl https://stream.nakji.network/sse?streams=nakji.common.0_0_0.market_trade-1m0s&namespace=sushiswap&subjects=<pairid>

wildcard streams​

# You can subscribe to all events in a stream name family.
# For example, for all Aave events you can subscribe to `nakji.aave.0_0_0.*`
wscat -c "wss://stream.nakji.network/ws?streams=nakji.aave.0_0_0.*"
curl https://stream.nakji.network/sse?streams=nakji.aave.0_0_0.*

multiple streams​

wscat -c "wss://stream.nakji.network/ws?streams=x&streams=y&streams=z"
curl https://stream.nakji.network/sse?streams=x&streams=y&streams=z

Notes​

Stream names are case sensitive. Pair IDs are case sensitive to their checksummed values.

For common streams, additional parameters namespace and subjects can be passed to further narrow down incoming messages.
