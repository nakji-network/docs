Overview

Stream name componentsâ€‹

Stream names are required to access and read data from Nakji. These names consist of 4 main parts:

  • Author: the person/organization that wrote the connector
  • Name: the name of the data source
  • Version: the version of the connector
  • Event: Event specifies the event that is submitted to Nakji Network by a source connector. For smart contract source connectors, the convention is to prefix the smart contract event with the smart contract name, eg. <smartcontractname>_<eventname>. For other source connectors, <category>_<eventname>. The category/smart contract name prefix may not be necessary in all cases.

There is an optional field that can be appended:

  • Aggregation Period: Appended using -. This describes the aggregation period.

In summary, a stream name is organized in this way:

Author.Namespace.Version.Smart Contract (or Category) _ Event (-Aggregation Period)

Examplesâ€‹

  • nakji.aave.0_0_0.aave_lpborrow
  • nakji.uniswapv3.0_0_0.pool_swap
  • nakji.common.0_0_0.market_openinterest-1m0s

When filtering, each segment can either be a specific filter to receive filtered messages or a wildcard (*) to receive all messages.
